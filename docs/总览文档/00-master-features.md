# 总功能设计文档（Master Features）

> 本文档是 interview-guide 项目的**产品总纲**，从用户视角描述整个产品做什么、有哪些角色、模块之间如何协作。
> 每个模块的详细功能设计见 `module-features/` 目录下对应文件。
> 技术实现见 `module-design/` 目录及 `00-master-tech.md`。
>
> **阅读顺序**：先读本文档建立全局认知，再按需深入各模块功能文档。

---

## 1. 产品定位

**AI 面试辅助平台**，服务求职者（应届生 + 3 年内经验）和导师两类用户。

一句话价值主张：**从简历诊断到模拟面试到知识巩固，一站式帮你拿到 offer。**

三大核心能力：
1. **简历智能分析** — 上传简历，AI 穿透式审计 + 结构化点评 + 重写范例
2. **AI 模拟面试** — Skill 驱动出题（11 个方向），文字 + 语音双模式，统一评估引擎
3. **知识库 RAG 检索** — 八股文/技术文档向量化检索，增强问答准确性

附加能力：语音实时面试、面试日程管理、多模型供应商管理。

## 2. 用户角色与权限体系

### 2.1 三类角色

| 角色 | 谁是 | 核心能力 |
|------|------|---------|
| **游客 GUEST** | 未登录访客 | 仅体验简历分析（每日 3 次，半遮面） |
| **STUDENT（求职者）** | 注册用户默认角色 | 全部业务功能 |
| **MENTOR（导师）** | 手动配置 | STUDENT 全部权限 + 可切求职者视角 |
| **ADMIN（管理员）** | 占位 | 第一版无后台，仅注解占位 |

### 2.2 游客 vs 登录用户权限矩阵

| 功能 | 游客 | 登录用户 |
|------|------|---------|
| 简历上传分析 | ✅ 每天 3 次 | ✅ 放宽（20 次/天） |
| 简历报告查看 | ⚠️ 半遮面（评分 + 前 3 条建议） | ✅ 完整报告 |
| 简历版本管理 | ❌ | ✅ 历史记录 + 评分趋势 |
| PDF 报告导出 | ❌ → 弹登录 | ✅ 每天 10 次 |
| 文字模拟面试 | ❌ → 弹登录 | ✅ |
| 语音面试 | ❌ → 弹登录 | ✅ |
| 知识库上传/管理 | ❌ → 弹登录 | ✅ |
| 知识库问答 | ❌ → 弹登录 | ✅ |
| 面试日程管理 | ❌ → 弹登录 | ✅ |

### 2.3 核心转化路径

```
游客访问 → 免费分析简历（半遮面）→ 想看完整报告 → 注册
                                            ↓
                                    解锁全部功能
```

**简历半遮面是唯一的引流入口**，其他所有功能锁登录。

## 5. 用户旅程地图

### 5.1 求职者完整旅程

```
1. 发现产品（首页）
   └─ 看到"免费分析简历" CTA

2. 体验简历分析（游客）
   ├─ 上传 PDF/Word
   ├─ 等待 6-16 秒
   ├─ 看到半遮面报告（评分 + 3 条建议）
   └─ 想看完整报告 → 点"注册"

3. 注册（手机号 + 密码）
   ├─ 填手机号 + 密码
   ├─ 报告自动补全（无缝接住）
   └─ 解锁全部功能

4. 深度使用（登录后）
   ├─ 简历：多次上传、版本管理、导出 PDF
   ├─ 面试：选方向（Java/阿里/字节...）→ 答题 → 看评分报告
   ├─ 知识库：上传八股文 → RAG 问答
   ├─ 语音面试：实时对话练习
   └─ 日程：粘贴面试邀约 → 自动解析 → 日历管理

5. 持续迭代
   ├─ 简历改了 N 版，看评分趋势
   ├─ 面试做了 N 场，看进步轨迹
   └─ 知识库积累，问答越来越准
```

### 5.2 导师视角

```
1. 登录导师账号
2. 可切换到求职者视角（纯 UI 切换，数据算自己的）
3. 体验全部求职者功能（用于教学演示/理解产品）
4. 导师专属功能（后定，可能包括：看学员简历、布置面试任务）
```

## 6. 全局产品决策（已锁定）

| 决策 | 选定 | 理由 |
|------|------|------|
| 游客身份 | deviceId（localStorage）| 免 cookie，省事 |
| 游客配额 | 3 次/天（仅简历）| 控成本 |
| 简历半遮面 | 评分 + 前 3 条，完整存 Redis（12h）| 引流 |
| 缓存有效期 | 12 小时 | 覆盖注册流程 |
| 登录方式 | 手机号 + 密码唯一 | 学习项目，砍验证码 |
| 忘记密码 | 砍掉 | 无验证码不安全 |
| 角色体系 | STUDENT/MENTOR/ADMIN | 导师可切视角 |
| 简历格式 | PDF + Word（图片砍掉）| 覆盖主流场景 |
| 多栏 PDF | PDFBox 检测 + Qwen-VL-OCR 降级 | 几何判断可靠 |
| 简历存储 | 原文加密进 MinIO + 结构化脱敏进 PG | 兼顾体验与安全 |
| 简历版本 | 多份独立（源项目模式）+ 评分趋势图 | 不做 diff |
| 简历去重 | SHA-256 哈希 | 省成本 |
| 文字面试 | Skill 驱动 + 并行出题（简历 60% + 方向 40%）| 照搬源项目 |
| 评估引擎 | 文字+语音共用 UnifiedEvaluationService | 复用 |
| 语音面试 | 千问 ASR/TTS/LLM 三件套 | 照搬源项目 |
| 知识库 | pgvector + 查询改写 + 动态阈值 | 照搬源项目 |
| 日程解析 | 三段式（规则正则优先，LLM 兜底）| 省成本 |
| 日历提醒 | 不做（源项目也没有）| — |
| Provider 切换 | 单例表 + registry.reload() | 热生效 |
| ApiKey 加密 | AES/GCM/NoPadding | 安全 |
| 异步任务 | 统一 Redis Stream | 复用基础设施 |

## 7. 全局约束（产品红线）

1. **成本红线**：游客每日 AI 调用上限 3 次，登录用户放宽但有上限
2. **安全红线**：ApiKey AES 加密，原文 PII 加密存储，JWT 鉴权
3. **体验红线**：所有耗时操作异步化（Redis Stream），用户不干等
4. **栈内红线**：不引入额外运行时（无 Python 脚本、无 Node 中间层）
